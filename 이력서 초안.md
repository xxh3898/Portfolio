### MediFlow
- 기간: 2025.10.20 ~ 2025.12.05 (약 6주)
- 인원: 6인 풀스택 팀 프로젝트 (W3C)
- 역할: DB 설계 담당 (DB Lead) / 근태 자동화 담당

#### 프로젝트 소개
병원 업무 효율화를 위한 ERP 통합 관리 시스템.  
의사·간호사·원무과 등 직군별 역할 기반 접근 제어와 Oracle DB 기반 복잡한 쿼리 최적화에 집중한 팀 프로젝트.

#### 담당 기능
- ERD 설계부터 DB 단 무결성 제약조건(`CONSTRAINT CHK_STATUS CHECK`) 설정 주도
- Oracle `LISTAGG`, 서브쿼리, View 활용으로 N+1 데이터 병합 및 쿼리 성능 향상
- LocalTime API로 출결 상태(정상/지각/조퇴/결근) 자동 판별 로직 구현

#### 기술 스택
- Backend: Java, Spring Boot, MyBatis
- Frontend: JSP, JSTL, jQuery, Axios
- Database: Oracle
- Tools: Git, GitHub, Notion

#### 기술적 문제 해결
**데이터 무결성 확보 및 DB 병합 충돌 해결**
- 문제 상황: 팀원별 상태 컬럼 기준 불통일로 DB 병합 시 데이터 충돌·정합성 오류 반복 발생
- 해결 방법: 상태 컬럼 타입 통일 후 DB 단 Check 제약 조건 강제 적용, 공통 더미 데이터 배포로 팀 환경 동기화
- 결과: 이후 DB 병합 충돌 0건, 팀 전체 개발 환경 동기화

### Cubing Hub
- 기간: 2025.10.02 ~ 2025.10.19 (약 3주)
- 인원: 1인 풀스택 개인 프로젝트
- 역할: 기획·설계·개발 전담

#### 프로젝트 소개
파편화된 큐브 기능(타이머·기록·알고리즘·커뮤니티)을 통합한 스피드큐빙 올인원 비동기 웹 플랫폼.  
타이머 누적 오차 제거와 렌더링 성능 최적화에 집중한 1인 기획/개발 프로젝트.

#### 주요 기능
- `Date.now()` Delta Time 방식 정밀 타이머 구현 (0.001초 단위)
- Stackmat 타이머 UX 이식 (idle → holding → running), WCA 규격 20-move 스크램블 생성
- VisualCube API 연동으로 CFOP 단계별 3D 알고리즘 시각화
- Zustand 도입으로 빈번한 상태 갱신 시 불필요한 리렌더링 최소화

#### 기술 스택
- Backend: Java 17, Spring Boot 3.4, Spring Data JPA
- Frontend: React (Vite), JavaScript ES6+, Zustand, Styled-Components, VisualCube API
- Database: MySQL, H2

#### 기술적 문제 해결
**정밀 타이머 누적 오차 문제 해결**
- 문제 상황: 브라우저 `setInterval` 특성상 밀리초 단위 누적 오차 발생, 실제 경기 수준 정밀도 미달성
- 해결 방법: `Date.now()` 기반 Delta Time 방식으로 타이머 코어 로직 리팩토링
- 결과: 0.001초 단위 실제 경기용 정밀 계측 구현
