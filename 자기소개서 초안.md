### 마스터 지원 동기 - 직무(백엔드 개발자) 공통 활용

CalmDesk 프로젝트에서 대시보드 API 25초 지연 문제를 겪으며, 애플리케이션의 비효율(N+1 쿼리)이 클라우드 인프라 자원(EC2 CPU 크레딧) 고갈로 이어진다는 점을 분석하고 해결했습니다. 이를 통해 시스템의 근본적인 병목을 인프라와 애플리케이션 양면에서 바라보고 최적화하는 백엔드 개발에 깊은 매력을 느꼈습니다.

(회사명)의 (구체적인 서비스/기술)처럼 많은 사용자 요청을 안정적으로 처리하는 시스템을 만드는 과정에 기여하고 싶어 지원하게 되었습니다.

## 2. 문제 해결 경험

CalmDesk 프로젝트에서 직원 대시보드 조회 시 25초에 달하는 지연을 발견하고, 애플리케이션 비효율이 클라우드 인프라에 미치는 영향을 분석해 근본적인 병목을 해결한 경험이 있습니다.

**문제 발견:**  
Chrome 개발자 도구를 활용한 응답 지연 확인 결과, 대시보드 조회 API에서 25초 지연을 확인했습니다. 이후 AWS Console 확인 결과, EC2(T3 Micro)의 CPU 크레딧이 고갈되어 스로틀링(Throttling) 상태에 진입한 것을 파악했습니다.

**원인 분석:**  
인스턴스 과부하 원인을 추적한 결과, 직원 데이터 조회 중 발생한 **N+1 쿼리 현상**(50명 조회 시 150개의 단건 쿼리 발생)이 DB 연산을 폭증시키고 인스턴스 CPU 자원을 과소비한 것으로 판명되었습니다.

**해결 방안:**  
150회의 단건 조회를 **`WHERE IN` 절을 활용한 벌크 쿼리 3회**로 변경했습니다. 이후 애플리케이션 내부에서 시간 복잡도 O(1)의 **`Map<Long, Entity>` 자료구조를 활용해 데이터 조립·매핑 프로세스**를 구축했습니다.

**결과/성과:**  
- **응답 속도 96% 향상:** 25초 → **0.9초 단축** (쿼리 호출 150회 → 3회)
- **안정성 복구:** 불필요한 연산 점유 최소화로 **EC2 CPU 사용률 정상화 및 CPU 크레딧 고갈 현상 해결**

**배운 점:**  
단순한 코드 레벨의 비효율이 어떻게 클라우드 인프라 자원의 마비로 직결될 수 있는지 체감하는 거시적인 병목 시야를 확립했습니다. 현재의 메모리 매핑 방식은 데이터 볼륨이 기하급수적으로 증가할 경우 애플리케이션 메모리 부하(OOM) 리스크가 존재함을 인지하고 있습니다. 이를 대비해 향후 트래픽 증가 시 페이징 처리 및 Redis를 활용한 글로벌 캐싱 구조로 전환할 수 있도록 결합도를 낮춘 설계를 고민하게 되었습니다.

## 3. 협업 경험

CalmDesk 6인 팀 프로젝트에서 이슈 관리자 역할을 맡아 팀의 협업 체계를 설계하고 운영했습니다.

**상황:**  
초기 개발 단계에서 팀원들이 커밋 메시지 형식이나 브랜치 네이밍을 제각각 사용해 코드 리뷰 및 변경 이력 추적이 어려웠습니다. 또한 PR에 설명이 부족해 리뷰어가 맥락을 파악하는 데 시간이 많이 소요됐습니다.

**본인의 행동:**  
팀 전체에 공통 적용할 Git 커밋 컨벤션(`feat`, `fix`, `refactor`, `docs` 등)과 GitHub 이슈 템플릿(기능 구현 / 버그 리포트), PR 템플릿(변경 사항 / 테스트 결과 / 관련 이슈 번호)을 직접 작성하여 팀에 배포하고 사용법을 공유했습니다.

**결과:**  
이후 팀원들의 커밋 메시지가 일관성을 갖추게 되었고, PR 설명 템플릿 도입으로 리뷰 소요 시간을 평균 30% 이상 단축했습니다. 팀장으로부터 "협업 체계 덕분에 후반 개발 일정 관리가 수월해졌다"는 피드백을 받았습니다. 최종 프로젝트 우수상 수상으로 이어졌습니다.

## 4. 입사 후 목표

**단기 목표 (입사 후 1~2년):**  
부하 테스트 도구(JMeter, K6 등)를 활용해 기존 및 신규 API의 트래픽 임계점을 수치화하겠습니다. N+1 문제 개선 경험을 살려 병목 구간을 선제적으로 파악하고, 쿼리 튜닝을 통해 서버 인프라 비용 대비 처리 효율을 높이는 백엔드 개발자가 되겠습니다.

**중기 목표 (3~5년):**  
트래픽·데이터 규모가 커져도 안정적으로 동작하는 서버 시스템을 설계할 수 있는 백엔드 엔지니어로 성장하겠습니다. 성능 병목 분석, 쿼리 최적화, 서비스 분리 설계 등 실무 경험을 쌓아 팀 내에서 기술적 의사결정에 기여할 수 있는 레벨을 목표로 합니다.

**기여 방식:**  
단순한 기능 구현을 넘어, 애플리케이션 코드와 인프라 리소스의 상관관계를 조망하는 병목 분석 시야를 바탕으로, (회사명)의 백엔드 시스템 안정성과 응답 속도 개선에 직접 기여하겠습니다.
