다중 접속 환경에서의 실시간 상태 동기화 및 멀티테넌시 아키텍처를 적용한 기업용 B2B HR·웰빙 SaaS 통합 관리 플랫폼.
다중 접속 환경에서의 WebSocket·SSE 기반 실시간 데이터 스트리밍과 멀티테넌시 아키텍처를 통한 기업 간 완전한 데이터 격리를 구현한 클라우드 구독형 서비스입니다.

주요 트러블슈팅
  • [근본 병목 개선] 애플리케이션 비효율로 인한 EC2 CPU 크레딧 고갈 대응 (응답 지연 25초 → 0.9초)
    1) 문제 발견
    Chrome 개발자 도구를 활용한 응답 지연 확인 결과, 관리자 대시보드 통계 조회 API(GET /api/dashboard/stats)에서 25초 지연.
    AWS Console 확인 결과, EC2(T3 Micro)의 CPU 크레딧이 고갈되어 스로틀링 상태 진입 파악.
    2) 원인 분석
    인스턴스 과부하 원인을 추적한 결과, 직원 데이터 조회 중 발생한 N+1 쿼리 현상(50명 조회 시 150개의 단건 쿼리 발생)이 DB 연산을 폭증시키고 인스턴스 CPU 자원을 과소비한 것으로 판명.
    3) 해결 방안
    JPA의 @Query JOIN FETCH를 활용해 연관 엔티티(Member)를 DB 계층에서 단일 벌크 쿼리(IN + JOIN FETCH) 1회로 한 번에 페치하도록 리팩토링. 이를 통해 150개의 단건 쿼리를 3개의 최적화된 벌크 쿼리로 통합했으며, 애플리케이션 단 Map 조립은 이미 DB에서 페치된 엔티티를 기반으로 추가 쿼리 없이 수행.
    4) 결과/성과
    • 응답 속도 96% 향상: 25초 → 0.9초 단축 (쿼리 호출 150회 → 3회)
    • 안정성 복구: 불필요한 연산 점유 최소화로 EC2 CPU 사용률 정상화 및 CPU 크레딧 고갈 현상 해결.
    • 이를 통해 애플리케이션 코드 스코프에서 벗어나, "앱 코드의 비효율이 어떻게 클라우드 인프라 자원의 마비로 직결될 수 있는지" 체감하는 거시적인 병목 시야 확립.
  • JWT & 프론트 통신 403 Error 해결
    1) 문제점 및 원인분석
    부서·근태 내역 조회 시 403 에러 반복 발생. 원인이 백엔드·프론트 양쪽에 분산되어 있어 특정이 어려운 상황.
    2) 개선/해결 방안
    프론트엔드 로컬 스토리지 키 불일치(token vs authToken)와 순수 Axios 직접 import로 인한 인터셉터 미적용 문제를 교차 분석해 원인 특정.
    3) 결과/성과
    키 이름 통일 및 공통 Axios 인스턴스 사용으로 수정 후 403 에러 완전 해소.
